# `humanctl` v1

## Thesis

`humanctl` is a local-first coordination layer for human-in-the-loop agent work.

It is optimized for one job: help agents avoid getting stuck on humans by packaging the right context, surfacing the smallest useful question, persisting the answer, and making future resumption cheap.

## Product principles

1. The shared store is the source of truth.
2. Live steer delivery is a convenience, not the only path.
3. Every visible object should be backed by files on disk.
4. HTML is a payload, not the architecture.
5. Agents should be able to write useful state with very few commands.
6. Humans should answer in the smallest possible unit of effort.

## Workspace model

`humanctl` uses a project-local `.humanctl/` directory by default.

```text
.humanctl/
  manifest.json
  inbox/
    events.jsonl
  tabs/
    main/
      manifest.json
      things/
  artifacts/
  state/
    ui.json
```

Optional global state can live in `~/.humanctl/`, but project-local state should be the default.

## Core objects

### Workspace

The top-level container tied to a directory.

Suggested fields:

```json
{
  "id": "workspace-local_git",
  "name": "local_git",
  "version": 1,
  "createdAt": "2026-03-14T00:00:00.000Z",
  "defaultTabId": "main"
}
```

### Tab

The main unit of navigation and focus.

Suggested fields:

```json
{
  "id": "main",
  "title": "Main",
  "description": "Primary shared work surface",
  "createdAt": "2026-03-14T00:00:00.000Z",
  "updatedAt": "2026-03-14T00:00:00.000Z"
}
```

### Thing

The atomic object an agent creates or updates inside a tab.

Examples:

- HTML preview
- markdown note
- image gallery
- request for input
- side-by-side comparison
- diff review
- live log view
- file or folder watch

Suggested fields:

```json
{
  "id": "hero-direction",
  "kind": "request",
  "title": "Pick the hero direction",
  "status": "open",
  "escalation": "ask",
  "render": {
    "type": "html",
    "entry": "content.html"
  },
  "attachments": [
    "preview-a.html",
    "preview-b.html",
    "notes.md"
  ],
  "needsResponse": true,
  "createdAt": "2026-03-14T00:00:00.000Z",
  "updatedAt": "2026-03-14T00:00:00.000Z"
}
```

### Event

The append-only queue that records meaningful changes and user responses.

Suggested fields:

```json
{
  "id": "evt_01",
  "ts": "2026-03-14T00:00:00.000Z",
  "kind": "answered",
  "target": {
    "tabId": "main",
    "thingId": "hero-direction"
  },
  "actor": "human",
  "payload": {
    "choice": "option-b",
    "comment": "The second one explains the point faster."
  }
}
```

## Escalation levels

V1 should support exactly four escalation levels:

- `log`: persist the update without interrupting
- `nudge`: lightweight interruption or resurfacing
- `ask`: explicit response requested
- `block`: work cannot continue without an answer

These levels should be reflected in both event metadata and UI treatment.

## Read state

Human attention state matters because agents should not thrash the interface.

Suggested UI state fields:

```json
{
  "thingId": "hero-direction",
  "seenAt": "2026-03-14T00:00:00.000Z",
  "readAt": "2026-03-14T00:00:00.000Z",
  "dismissedAt": null,
  "closedAt": null,
  "pinned": false,
  "changedSinceSeen": true
}
```

## CLI shape

V1 CLI should stay small and obvious:

- `humanctl init`
- `humanctl status`
- `humanctl tab create`
- `humanctl thing put`
- `humanctl event append`
- `humanctl ask`
- `humanctl focus`
- `humanctl watch`

Implemented in this repo snapshot:

- `humanctl init`
- `humanctl status`
- `humanctl ask`
- `humanctl serve`

## Rendering model

The host app should treat HTML as one render target among several:

- `html`
- `markdown`
- `image`
- `diff`
- `file`
- `gallery`
- `form`
- `stream`

The app owns layout and persistence. `Thing` payloads provide content.

## Why this model

This design keeps the system durable and agent-friendly:

- file-backed state survives chat threads
- append-only events enable both live and async responses
- HTML remains easy for agents to generate
- stable object IDs make resumption possible
- the app can stay simple while supporting rich media
