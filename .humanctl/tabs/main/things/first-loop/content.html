Decision packet

What should humanctl optimize for first?

The infrastructure is live. The first product decision now is behavioral, not technical: what kind of human unblock loop should the app make cheapest on day one?

Fast answers

Start with the sharpest path from blocker to answer. Surface one decision, collect one response, resume immediately.

Ambient ops

Start with the inbox and queue. Make it obvious what changed, what is blocked, and what the human has not seen yet.

Artifact studio

Start with previews, diffs, and rich media. Make the product a better place to show work before optimizing response loops.

The recommended path is Fast answers. It is the most opinionated version of the thesis and the cheapest route to proving that agents can package blockers better than chat threads can.
