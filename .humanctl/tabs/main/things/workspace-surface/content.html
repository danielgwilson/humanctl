Prototype note

Build the surface against real files.

The first useful app view should not invent app-only state. It should read the local workspace, render the tabs and Things, and make the inbox legible enough that a new session can resume without replaying the thread.

  • Read tabs, Things, and events from .humanctl/.
  • Make one selected Thing the focus, with the queue visible next to it.
  • Let an open ask persist an answer back to disk.
